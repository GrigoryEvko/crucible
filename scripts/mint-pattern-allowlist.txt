# scripts/mint-pattern-allowlist.txt — §XXI scanner grandfather list.
#
# Two interchangeable entry grammars are honoured:
#
#   NAME-KEYED  path:mint_name[:check-ok]   (PREFERRED — drift-proof)
#   LINE-KEYED  path:line[:check-ok]        (legacy — fragile)
#
# where <check> ∈ {nodiscard, noexcept, constexpr, requires}.  Lines
# beginning with '#' are comments.  A bare key (no `:check-ok` suffix)
# exempts all four checks for that key.
#
# ── Why name-keyed is the default ────────────────────────────────────
#
# A LINE-KEYED entry pins an exemption to a source LINE NUMBER.  Any
# edit above the mint — a concurrent agent inserting a member, a new
# `#include`, a reflowed comment — shifts the line and silently
# invalidates the entry: the real mint reads as an un-suppressed
# violation AND the now-dangling entry trips the stale-entry gate.  That
# is the exact failure mode this file kept hitting (the FIXY-U-101
# line-collapse sweep stranded 7 entries; later sweeps stranded more).
#
# A NAME-KEYED entry pins the exemption to the mint's IDENTIFIER (the
# `mint_<noun>` token on the candidate line).  The identifier is stable
# across line shifts, so a name-keyed exemption keeps tracking the same
# factory no matter how the file above it churns.  This is the fix-18
# approach applied to the mint guard: eliminate the line number as the
# drift vector entirely.  Every entry below is name-keyed.
#
# Name-keyed semantics: `path` must match the candidate's file exactly;
# the name segment matches the `mint_<noun>` identifier.  An overload
# SET sharing one name in one file collapses to a single entry — that is
# intentional: an overload set is one §XXI factory with one carve-out
# rationale (e.g. `mint_persisted_session` has three live overloads, all
# allocating, one `:constexpr-ok` entry covers them).
#
# ── Trim discipline ──────────────────────────────────────────────────
#
# Every entry is either a STRUCTURAL exemption (allocating / syscall /
# token / concept-constrained-param / passkey ctor) OR a tracked
# migration TODO.  When a site migrates to canonical §XXI shape (or the
# scanner learns to recognize a new structural false-positive), the
# entry MUST be removed; the stale-entry gate fails CI if a dead entry
# lingers.  The goal is for this file to trend toward the
# structural-exempt minimum.
#
# ── Carve-out categories ─────────────────────────────────────────────
#
#   `constexpr-ok` — allocating / syscall / resource-synthesizing mint
#     (or a class-method mint reading runtime state).  §XXI rule (3)
#     explicitly permits dropping `constexpr` "in favour of plain
#     [[nodiscard]] noexcept" for genuinely-allocating mints.  Examples:
#     fixy/Mmap.h `mmap(2)`, fixy/Io.h `io_uring_setup(2)`, the eBPF /
#     perf-event `perf/*` mints, the `canopy/*` gossip-state mints, the
#     class-method view mints (`mint_open_view`, `mint_compiled_view`).
#
#   `requires-ok` — a templated mint whose §XXI gate is expressed as a
#     concept-CONSTRAINED template parameter (`template <SomeConcept T>`)
#     or an in-body `static_assert(...)` rather than a trailing
#     `requires` clause.  The scanner only recognizes the literal
#     `requires` keyword, so these read as "missing requires" though the
#     gate is real.  Also covers TOKEN mints (§XXI: token mints derive
#     authority from a parent token, NOT a `CtxFitsX` concept gate —
#     `mint_permission_root` / `_split` / `_combine` / `_split_n` /
#     `_combine_n` / `_share`).
#
#   `noexcept-ok` — a mint being migrated to noexcept, OR (in a BANNED
#     directory the guard may not edit) a real omission that can only be
#     remediated by an allowlist entry.  See cntp/AfXdp.h below.
#
#   `nodiscard-ok` — a mint whose return value can legitimately be
#     discarded, or a passkey-ctor false positive.
#
# Cross-references:
#   fixy-M-22 (#1507) — CLOSED: mint_vigil_mode_bridge gained constexpr.
#   fixy-M-23 (#1508) — CLOSED: mint_persisted_session gained noexcept on
#       all three live overloads.  constexpr-ok carve-out retained.
#   fixy-A4-027 (#1666) tracks mint_federation_admittance nodiscard drift.
#
# NOTE for human review — real omissions in BANNED directories:
#   cntp/AfXdp.h mint_af_xdp_socket genuinely SHOULD be `noexcept` (it
#   forwards to allocating XDP-socket setup with no documented throw),
#   but cntp/ is a guard-edit-banned tree, so the only available
#   remediation is the `:noexcept-ok` entry below.  Fix the source when
#   the cntp/ edit ban lifts and remove the entry.
