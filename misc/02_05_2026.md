## Sequencing decision

1. **Now → ~4 weeks: Phase 0.** Ship P0-1 through P0-6. No other backlog work in flight unless it directly unblocks Phase 0.
2. **Phase 0 close → ~12 weeks: Bucket 1, dependency-ordered.** Start with SEPLOG-I1 → I2/I3/I4/I5/I6 → SEPLOG-K1..K4 → SAFEINT-Epoch-I → METX-AUDIT/VERIFY → FOUND-G/I/K substrate-resident sites.
3. **In parallel with Bucket 1: Bucket 2 hardening.** Independent; can be picked up as one-shot tasks between Bucket 1 milestones.
4. **Bucket 3 stays parked** until the architectural move (`fixy/` directory created, headers physically relocated). Then re-target en masse and resume.
5. **Bucket 4 is closed** by either deletion (obsolete) or migration-piggyback (superseded).

The Phase 0 commitment is structural: every Bucket 1 task references the substrate Phase 0 builds, and every Bucket 3 task wraps headers that compose against Phase 0's Fn template. Doing them out of order means rework.
